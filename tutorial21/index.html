CSS Specificity

The Cascade Algorithm

The cascade algorithm in CSS is used to solve conflicts when a same Html element styled by more than one properties. Here are some points of cascade algorithm:
Position and order of the appearence:
If same type of selector(e.g class) is used to give different properties then the selector which is at last will be considered.
Specificity:
An algorithm that determines which CSS selector has the priority. Here is the order of specificity:
Inline Style > Id Selector > Class or attribute selector > Element Selector > Universal Selector
Origin:
The order in which CSS appears and where it comes from. There are some styles which comes from the browser. These styles are coming from user-agent style sheets. So the styles which we write ourself has high priority than the one applied by the browser.
Important:
If we add an important keyword in any element then it has the highest priority. It will have more priority than inline CSS.

Specificity Calculations:

To calculate specificity, assign a value to each part of the selector.

  • Universal Selector - 0
  • Element and pseudo element - 1
  • Class selectors, attributes selectors or pseudo class - 10
  • Id Selectors - 100
  • Inline Selectors - 1000
